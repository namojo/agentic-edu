# 커리큘럼 설계 스킬

`curriculum-architect` 에이전트가 사용하는 도구. 4시간짜리 "Agentic AI 개발 1회차" 교안의 골격을 만든다.

## 왜 이 스킬이 필요한가

비전공자가 섞인 교육은 "이론 → 실습"의 선형 구조에서 쉽게 이탈한다. 이 스킬은 모듈을 30~60분 단위 완결형으로 쪼개고, 각 모듈 끝에서 "만들어봤다"는 성취를 남기도록 강제한다. 또한 전체 수업의 두 축을 **실습 A(파일 관리 MCP+Skill)**와 **캡스톤 B(뉴스 멀티에이전트)**로 고정해, 학습자가 배운 개념이 어디서 쓰이는지 끊임없이 보이게 한다.

## 설계 절차

1. **대상·환경 고정**: 삼성SDS 직원(비전공자 포함) + Cursor + Gemini 무료 API. 이 전제를 벗어난 실습은 즉시 탈락.
2. **모듈 7개 골격 확인** — 순번·시간·난이도 라벨 필수:
   1. 바이브코딩 (30분/입문)
   2. Cursor 투어 (30분/입문)
   3. Skill 만들기 (40분/초급) — 실습 A 전반
   4. MCP 만들기 (40분/초급) — 실습 A 후반
   5. 단일 Agent (40분/중급) — 보너스 A1
   6. Multi-Agent (40분/중급) — 보너스 A2
   7. 캡스톤 뉴스 (60분/중상) — 실습 B + 심화 과제 C1~C3 소개
3. **각 모듈에 4개 필드**: `theory`, `hands_on`, `learningOutcomes`, `keyVisualPrompt`
4. **실습 A/B 메타데이터** 분리 기록 — 어느 모듈과 엮이는지, 최종 산출물이 무엇인지
5. **메뉴 구조** 확정 — 개요 / 커리큘럼 / 실습 / 슬라이드 / 자료 5개 상단 메뉴

## 출력 스키마

`_workspace/01_curriculum.json`은 `agents/curriculum-architect.md`의 "출력" 섹션 스키마를 그대로 따른다. 이 스킬 본문에서는 스키마를 중복 선언하지 않음.

## 난이도 곡선 판별 기준

- 첫 2개 모듈은 "코드 한 줄 안 써도 따라갈 수 있는지" 점검
- 3~4번 모듈은 "붙여넣기 + 클릭으로 90% 해결"되는지
- 5~6번 모듈은 "비전공자가 개념 이해까지 도달 가능한지"
- 7번 모듈은 "전공자가 심화하면 재밌는지"

## 비유 사전 (고정 용어)

| 기술 용어 | 교안 내 비유 |
|----------|-------------|
| Skill | 팀원에게 주는 업무 매뉴얼 |
| MCP | AI를 위한 USB-C |
| Agent | 전담 업무 비서 |
| Multi-Agent | 여러 비서로 구성된 소규모 팀 |
| Orchestrator | 팀장 |
| Workflow | 정해진 순서대로 여러 사람이 릴레이하는 업무 |

이 사전은 `_workspace/GLOSSARY.md`로 나중에 export되어 전 모듈의 용어 통일에 쓰인다.

## 재호출 시 동작

- 사용자가 "모듈 N 내용 늘려 / 줄여" → 해당 모듈의 `duration`과 `hands_on` 수정
- 사용자가 "실습 추가" → `practices` 블록에 새 키 추가, 관련 모듈의 `hands_on`에 cross-link
- 사용자가 "일본 뉴스 Agent 추가" → `practices.B_news_multiagent.agents`에 추가 + 모듈 07 설명 갱신
