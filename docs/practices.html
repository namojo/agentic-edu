실습 A는 "Skill = 매뉴얼, MCP = 도구"의 분리 감각을, 캡스톤 B는 4개 에이전트 협업의 오케스트레이션 감각을 책임집니다.

A
실습 A · 모듈 03 + 04

파일 관리 MCP + Skill 만들기

로컬 파일 시스템 대상. 하나의 자연어 요청("다운로드 폴더 정리해줘")으로 Skill이 매뉴얼을 펴고 MCP가 실제 이동을 수행하는 흐름을 완성합니다.

1
Skill 작성
file-organizer/SKILL.md
2
Cursor 등록
%USERPROFILE%\.cursor\skills\
3
대화로 정리
"Downloads 폴더 정리해줘"
4
한계 발견
Skill만으론 실행 불가
5
MCP 연결 · 재시도
file-manager-mcp + mcp.json
Skill

매뉴얼 — 지식 · 절차

  • SKILL.md 1장 (markdown + frontmatter)
  • description의 트리거 표현에 매칭되면 자동 로드
  • 저장 위치: %USERPROFILE%\.cursor\skills\file-organizer\SKILL.md (macOS: ~/.cursor/skills/...)
  • 실제 파일 이동·삭제 같은 "실행"은 불가
MCP

도구 — 실행권 · 손과 발

  • JSON-RPC 2.0 기반 표준 프로토콜 (스펙 2025-11-25)
  • Host(Cursor) ↔ Client ↔ Server(능력 제공자)
  • 등록 위치: %USERPROFILE%\.cursor\mcp.json (macOS: ~/.cursor/mcp.json)
  • 실제 파일 이동·쓰기·검색을 실행 (공식 server-filesystem 또는 직접 만든 file-manager-mcp)
%USERPROFILE%\.cursor\mcp.json — 공식 filesystem 서버 등록 (Windows 기준)
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Users\\<사용자>\\Downloads"
      ]
    }
  }
}

이 한 줄이면 끝. Cursor를 재시작하고 Settings → MCP에서 filesystem 도구 목록(list_directory, move_file, search_files…)을 확인합니다. macOS에선 /Users/<me>/Downloads 경로 사용.

B
캡스톤 B · 모듈 05 ~ 07

한 / 미 뉴스 멀티에이전트

키워드 1개 입력 → 병렬 수집 → 팩트체크 → 통합 리포트. 4개 에이전트와 파일 MCP 재사용, 공용 워크스페이스 패턴을 모두 체험합니다. 작업 폴더: news-multiagent/.

kr-news-agent

한국 뉴스 수집 · 1차 요약

네이버/다음 RSS 또는 공식 API에서 키워드 관련 상위 5개 기사를 가져와 헤드라인·핵심 수치를 정규화. 산출: _workspace/kr.json

us-news-agent

미국 뉴스 수집 · 1차 요약

Google News RSS 등 공개 소스에서 상위 5개 수집 후 kr과 동일한 스키마로 정규화. 산출: _workspace/us.json

fact-checker

양측 교차검증 · 불일치 플래그

kr/us 두 파일을 로드해 주장·수치·인용문을 매칭하고 최소 2개 독립 출처 교차 규칙을 적용. 산출: _workspace/facts.md

orchestrator

팀장 · 병렬 실행 · 통합 리포트

kr/us 병렬 호출 → fact-checker 릴레이 → 템플릿 주입으로 reports/YYYY-MM-DD.md 생성. 재시도·중복 제거 책임.

심화 과제 (아이디어)

수업이 끝난 뒤, 여기서부터

C1~C3은 "수업에선 다루지 않지만 혼자 확장해볼" 방향입니다. 캡스톤 B를 완성한 뒤 응용해 보세요.

C1

일본 뉴스 Agent 추가

jp-news-agent를 병렬 라인에 붙여 3개국 비교 표로 확장. orchestrator의 fan-out 리스트에 한 줄 추가하면 됩니다.

C2

Playwright로 원문 URL 검증

fact-checker 단계에서 실제 페이지를 헤드리스로 열어 타이틀·본문 존재 여부 확인. MCP의 playwright 서버를 사용합니다.

C3

Slack 웹훅으로 리포트 배달

orchestrator 종료 훅에 incoming webhook POST를 추가. 매일 오전 9시 채널에 자동 발송되는 뉴스 브리프가 완성됩니다.
